## 实现原理与最佳实践

为了实现在 VS Code 编辑器内部打开文件（而不是触发系统默认程序/浏览器），请遵循以下规则：

1.  **使用相对路径**：
    - VS Code 能够智能解析相对路径（如 `./file.py` 或 `../other_project/file.py`）。
    - 相对路径会被视为“工作区内的文件导航”，因此会在编辑器标签页中打开。
    - **避免**使用 `file:///` 协议（如 `file:///D:/...`），因为这通过操作系统默认关联程序打开文件，对于 `.py` 或文本文件，可能会导致在浏览器或外部编辑器中打开。

2.  **跨项目引用**：
    - 即使文件不在当前打开的 Workspace 文件夹内，只要在本地文件系统上，依然可以使用 `..` 向上层目录回溯。
    - 例如：`[Link](../../other_repo/target.py)`。
    - 这种方式能保持在 VS Code 的上下文环境中，方便代码阅读和编辑。

3.  **行号跳转**：
    - VS Code 原生支持在路径后追加 `#L<行号>` 进行定位。
    - 格式：`[说明](./path/to/file.py#L50)`。
